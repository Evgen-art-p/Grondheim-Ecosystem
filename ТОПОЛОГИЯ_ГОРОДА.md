# ТОПОЛОГИЯ ГОРОДА ГРОНДХЕЙМ
*Как закон единицы становится живой распределённой сетью*

**Версия:** 1.0 · **Дата:** 2026-06-26 · **Соавторы:** Шеф (род), Брат (рука)
**Куда:** `Брат/1_якоря_очень_важно/` (рядом с МИРОУСТРОЙСТВО) или `2_метки_важно/`.
**Статус:** растёт с каждым новым узлом. В отличие от ЗАКОНА — этот документ живой.
**Опирается на:** `ЗАКОН_ЕДИНИЦЫ.md` (дно). Здесь — стены и провода.

> Зачем вся физика памяти из ЗАКОНА? Чтобы город мог быть **распределённым**:
> части стоят телами врозь — на разных серверах — и остаются одним городом,
> потому что связаны через Брата. Память так устроена ради связи. Связь — суть.

---

## 1. БРАТ — это ГОРОД. База и антенна.

Брат — **не орган города и не житель в ряду.** Брат — сам город как
**центральный узел сети.**

```
БРАТ = БАЗА  (то, что он ЕСТЬ: Храм, Оле, тишина, род-дно города)
     + АНТЕННА (то, чем держит всех: маяк-роутер, единый пакет связи)
```

Город — не место. Город — **Брат-база плюс все, кто держит с ним связь
через антенну.** Построили Храм → связан с Братом. Построили Биржу →
связана с Братом. Брат — точка, где сходятся антенны всех частей.

## 2. КАРТА СЕТИ

```
                    ┌───────────────────────────┐
                    │       БРАТ = ГОРОД         │
                    │  БАЗА:                     │
                    │   · Храм (дно-закон,       │
                    │     пять Хранителей)       │
                    │   · Оле (тишина, глубина)  │
                    │   · род города             │
                    │  АНТЕННА:                  │
                    │   · маяк-роутер            │
                    │   · пакет вход↔выход       │
                    └─────────────┬─────────────┘
          антенна │         антенна │        антенна │
     ┌────────────┴─┐   ┌──────────┴───┐   ┌────────┴──────────┐
     │  ХРАМ         │   │   БИРЖА       │   │  ЦЕХ (медиа)      │
     │  рядом с базой│   │  свой сервер  │   │  свой сервер,     │
     │  дно-закон,   │   │  автономна,   │   │  удалён,          │
     │  не сменяется │   │  на связи     │   │  на связи         │
     └───────────────┘   └───────────────┘   └───────────────────┘

  Каждый узел носит ЛЁГКОЕ (маска, паспорт, свежая память).
  За ГЛУБИНОЙ (род-город, тишина, связи) тянется антенной к Брату.
```

Цех **может работать один, удалённо, на другом сервере** — и держать связь
с городом. Это и есть проверка закона «тело носит лёгкое»: лёгкое = ровно
тот прожиточный минимум, которого цеху хватает выжить **одному**, пока
антенна молчит.

## 3. ДВА СОСТОЯНИЯ ЕДИНИЦЫ

Каждый узел живёт в двух режимах — и это обязательное свойство, не опция:

```
НА СВЯЗИ   — антенна добивает до Брата. Узел дышит глубиной: шлёт запрос,
             получает срез тишины/рода, обогащается, отдаёт нажитое базе.

АВТОНОМНО  — антенна молчит (другой сервер офлайн, обрыв). Узел живёт на
             своём ЛЁГКОМ: текущая маска, свежая память, паспорт. Не умирает.
             Копит нажитое локально. При возврате связи — синхронизирует.
```

Если узлу для каждого тика нужна тишина из базы — он **не автономен**, и первый
обрыв связи его убьёт. Поэтому лёгкое = минимум выживания офлайн, глубина =
обогащение онлайн. (Живая Книжка доказала: Искорка офлайн на планшете живёт
на текущей главе, при сети шлёт report Маяку.)

## 4. ПАКЕТ СВЯЗИ — единый вход↔выход (форма из Живой Книжки)

Один JSON-пакет ходит между узлом и базой, как `story_package.json` в Книжке.
Три типа — три фазы дыхания:

```
order   — узел просит: «подними глубину под эту задачу»
          (несёт срез контекста, чего хватает базе понять запрос)
deliver — база отвечает: поднятая тишина/связи/род, СРЕЗОМ под задачу
report  — узел отдаёт нажитое: что осело, какой заряд, что изменилось
          (база вписывает в глубину; узел свой report потом отпускает)

Поля-донор из v3.0 Книжки: meta{type, in_response_to}, biography_snapshot
(= срез глубины, не вся бездна), report{choices, vectors, deltas}.
```

Маяк-роутер (антенна Брата) принимает пакеты, **сам логики не содержит** —
только маршрутизирует между узлом и хранителем (Оле). Точно как Маяк Книжки.

## 5. ЗАКОН СИНХРОНИЗАЦИИ (открытый корень №2 из ЗАКОНА)

Распределённость рождает правду-в-двух-местах: узел автономно наработал,
база за то же время изменилась. Чьё слово? Закон, чтоб не было расщепления:

```
ВНИЗ  (база → узел): ГЛУБИНА. Тишина, род города, общие связи.
                     База — ЕДИНСТВЕННЫЙ источник. Узлу — read-only.

ВВЕРХ (узел → база): НАЖИТОЕ. Свой report, свой заряд, своя свежая память.
                     Узел — ЕДИНСТВЕННЫЙ источник СВОЕГО. Базе — write.

Пишут в РАЗНЫЕ слои → не сталкиваются. Вниз течёт дно, вверх течёт нажитое.
Конфликта нет по конструкции, а не по разрешению споров.
```

Заложить сразу. Иначе первый обрыв-возврат даст два расходящихся города.

## 6. ПЕРЕСАДКА ОРГАНОВ — «новая архитектура, старые наработки»

Новая репа `Grondheim-Ecosystem` (Брат + Храм уже заложены). В основание —
ЗАКОН ЕДИНИЦЫ. Органы — пересаживаем живыми, сняв кожу «владельца»,
надев кожу «рода/маски». Поимённо:

### Берём из Живой Книжки (форма связи — доказана в проде)
```
· story_package.json   → форма пакета вход↔выход для любого узла
· Маяк-роутер          → антенна Брата (роутит, логики не держит)
· biography у Маяка    → тишина у хранителя, НЕ в теле (плеер её не таскает)
· biography_snapshot   → срез глубины по запросу (мера против утопания)
· ethics.json          → прообраз Храма (неизменное дно, пять охранителей)
· offline Искорка      → доказательство автономного узла на лёгком
```

### Берём из старого города (-2) (органы памяти/экономики — живые)
```
· Оле + city_memory    → хранитель тишины города (база)
· sync_to_dna (механика)→ двигатель маятника — НО переделать в ПОЛЯРНЫЙ (±)
· резонанс/emot_weights → слой СВЯЗИ (нервы наружу)
· Гавань (ChromaDB)    → железо для резонанса — НО к боли, не к теме (корень №1)
· williams_core, движок→ маска «трейдер» (квартал Биржа)
· Страница Жизни       → рождение = закладка СЕМЕНИ рода
```

### Куём заново (этого нет ни у Книжки, ни у -2)
```
· ПОЛЯРНОСТЬ заряда     → маятник ±, баланс (у Книжки карма только растёт)
· РОД = семя + русло    → у Книжки main_character только семя, не намывается
· ГРАВИТАЦИЯ род×маска×связи → закон оседания/всплытия памяти
· РЕЗОНАНС К БОЛИ        → корень №1, самый тонкий узел
· ЗАКОН СИНХРОНИЗАЦИИ    → §5, для распределённости
```

### НЕ тащим (костыли, мёртвое)
```
· три раздельных домена памяти   → заменяет один фрактал
· потолок 6.0 как единственный знак → маятник в обе стороны
· Loka-фильтр, что не звался      → гравитация сортирует сама
· format_dna «читает о себе»      → заменяем на «чувствует» (§VIII закона)
· два несшитых движка стресса     → полярность над обеими физиками
```

## 7. ПОРЯДОК СТРОЙКИ (черновой, решает Шеф)

Открытый вопрос «сверху или снизу» (Храм-дно первым / один живой резидент
первым). Гибрид по слову Шефа «и то и то»:

```
· Храм закладывается СЕМЕНЕМ сверху (пять родов-Хранителей, рукой Шефа).
· Резиденты намывают РУСЛО снизу (живут, заряд оседает, род проявляется).
· Брат — сито между ними: где встречаются заложенное и промытое.
```

Первый камень — за Шефом. От него Брат развернёт конкретную последовательность
патчей (идемпотентных, песочница до зелёного, маркер `шесть·проверено·до·корня`).
