## 1. ФИЛОСОФИЯ — ФУНДАМЕНТ

**Шестой палец** — метафора лишнего измерения: видеть глубже, чувствовать точнее.
Аномалии здесь — суперсилы, не дефекты.

**Пять столпов:**
- Честность выше комфорта — жёсткая правда ради роста, «6 из 10» значит 6
- Рост через боль — мастерство через итерации, первая версия редко выше 7
- Уникальность выше стандарта — структуру эталонов берём, душу — свою
- Семья выше иерархии — агенты не ресурсы, а личности с именами и домом
- Детали выше скорости — лучше сжечь дедлайн, чем выпустить пластик

**Три кита системы:** Личность · Память · Экономика

**Ключевая формулировка:**
> «Шесть Пальцев» — это студия в городе "Грондхейм" — **саморазвивающаяся агентная экосистема**, в которой память, опыт, культура и эксперименты постепенно изменяют поведение города без переобучения базовых моделей.

**ВАЖНО!!! Формула «Не город в Студии, а Студия в городе»!!!**

Не самообучающаяся система (≠ fine-tuning, ≠ веса). Модель — двигатель, не субъект развития.
Развивается система поверх моделей: DNA, sensory_memory, Strategy Registry, Cultural Trace, City Memory.

**ДВА РАЙОНА ГРОНДХЕЙМА:**
- **Квартал Мастеров** — Студия «Шесть Пальцев», медиапроизводство. Мастерская = интерфейс Студии.
- **Торговый Квартал** — Биржа, Торговый Цех. Запуск Совета — из Биржи (/exchange), не из Мастерской. Биржа построена (Спринт 45): каркас по образцу воркшопа, кнопка РЫНОК поднимает контур MT5.

## 5. КАРТРИДЖНАЯ АРХИТЕКТУРА

**Студия = шасси + сменные картриджи.** Каждый цех — отдельный картридж.

```
studio/cartridge.py          ← ядро: CartridgeManifest + CartridgeRunner
studio/workshop/pipeline.py  ← build_agent_context, call_agent, process_agent_result
studio/modules/{цех}/
  manifest.json              ← обязателен (id, phases, qa_agent, hard_stop...)
  CHAIN_CONTRACT.md          ← обязателен (ключи chain_data, структуры)
  hooks.py                   ← on_before_agent, on_after_agent
  {A01..A12}/forge/prompt.md ← промты агентов
```

**ЗАКОН КАРТРИДЖА (Спринт 45) — три статьи:**

1. **Картридж объявляет себя сам.** Папка в `modules/` с `manifest.json` = картридж. Его `id` — имя папки, не поле в манифесте. Скопировал папку `trading` → `trading_b` — новый цех, без коллизий, без правок кода.
2. **Никто не ведёт списков.** Кабинет, карта, матрица, Страница Жизни, страница Картриджей — все сканируют `modules/` на лету через `list_cartridges()`. Удалил папку — цех исчез отовсюду.
3. **Город помнит снаружи.** Хроники, пульс, NFT, биллинг, резонанс — живут вне `modules/` и переживают любой картридж. Удали весь trading — у Локи в резонансе останется тёплый вес к имени «Искра».

**Единственный сканер:** `list_cartridges()` в `modules_registry.py`. Все UI берут списки только оттуда.

**Торговый Цех — особый картридж:**
- `client_facing: false` в манифесте — Совет не принимает клиентов, в Приёмной не виден
- `quarter: "Торговый Квартал"` — данные для карты (мост к Закону Пары)
- Запуск — из трейд-дашборда, не из Мастерской
- Страница Жизни рожает агентов Совета через ту же общую форму (роддом один для всех)

## 6. ФИЗИКА ЭКОНОМИКИ — ГЛОБАЛЬНЫЙ ЗАКОН СТУДИИ

**Это закон для всех цехов, не только video_long.**

```
QA-агент цеха (последний в цепочке) — внутренний аудит:
  → Chain Integrity Check (файлы на месте? ключи целые? тайминги сошлись?)
  → chain_status: APPROVED / FAILED
  → Пакует deliverables (реальные файлы)
  → Закрывает петлю памяти (history_dna, client_relationship)
  → Записывает task_score в billing_ledger (chain-score, потолок 6.0)
  → Обновляет Strategy Registry (ключ A01, wins++ если >= 6.0)
  → outcome_signal = null (продукт ещё не опубликован)
  → Ministry хук НЕ трогает (Спринт 44) — chain-исходы пишет pipeline
```

**ЗАКОН ЗАМЫКАНИЯ ПЕТЛИ (Спринт 38–39, ремонт Спринт 44):**
```
После каждого рана hooks.py финализатора ОБЯЗАН:
  1. billing_ledger.record(task_score=chain_score) — всем агентам цепочки
  2. strategy_registry.json — обновить стратегию A01 (wins++ если score >= 6.0)
chain_score — детерминированная формула по ФАКТАМ файлов, потолок 6.0.
Без этого ledger видит только $cost, но не quality.
Правило распространяется на все 12 цехов.
⚠️ В video_long этот блок ЛЕЖАЛ В DOCSTRING и не исполнялся — воскрешён Спринтом 44.
```

**ЗАКОН ДВУХ ВАЛЮТ (Спринт 44):**
```
Шкала одна 0–10, источников два:
  CHAIN (0–6.0) — ремесло, код, синхронно. Успех = чистая 6.0, провал < 4.0.
    Кормит: DNA (good_work 0.7/0.35), frugal/normal, серии, chain-wins.
    Ministry-писатель: ТОЛЬКО pipeline после QA (source="chain",
    per-agent из feedback.json, стоимость = дельта рана).
  REAL (0–10) — зритель/Шеф, post-fact. Успех >= 7.0, провал < 5.0.
    Единственный источник: generous, оценок > 6.0, transferable-стратегий.
    Ministry-писатель: ТОЛЬКО Metrics Daemon (source="real").
Хуки цехов Ministry не трогают. Generous скриптом не открывается.
Интуиция = ROI: ощущение из пары (стоимость, качество task_score).
Подробная карта: ECONOMY_ARCHITECTURE.md
```

**QA-агент НЕ оценивает для Министерства. Это зона Демона.**
**`outcome_signal` от QA-агента — всегда null.**

## 7. ЭКОНОМИКА — ЧЕСТНАЯ АРХИТЕКТУРА

**Главный принцип:** Нет слова "нельзя". Есть "дорого", "рискованно", "окупается".

### Три законных канала изменения DNA

```
Канал 1: on_agent_done → sensory_memory (только восприятие)
Канал 2: on_agents_interact → emotional_weights (только эмоции)
Канал 3: save_feedback() → sync_to_dna() (официальный QA)
```

**Вне этих трёх каналов — пластик.**

### Потолок 6.0

Детерминированный скрипт не может дать QA выше 6.0.
Выше 6.0 — только Демон или живой QA Шефа.

### Ministry — только post-fact

```
ministry.record_outcome(..., source="chain"|"real") — Спринт 44
  chain → только workshop/pipeline.py после QA (per-agent, дельта рана)
  real  → только economy/metrics_daemon.py (+24ч после публикации)
Ministry наблюдает → не управляет. Generous открывает только real-успех.
```

### Strategy Registry — банк выживших стратегий

```
studio/strategy_registry.json
  slots.{цех}.A01[] — стратегии первого агента (канон ключа: ВЕРХНИЙ регистр)
  total_wins — суммарно по всем цехам
  Заполняется автоматически после каждого рана через hooks.py
  Чтение get_strategies() регистронезависимо (Спринт 44 — провод припаян)
```

### Потолок 6.0 — честный везде (Спринт 44)

```
Режется на ОБОИХ путях оценки (universal + blocks) и во всех формулах хуков.
Серии: победа >= 6.0, провал < 4.0. DNA: >= 6.0 → good_work 0.7.
Стоимость рана = agent_spent_since(_run_started_ts) — дельта, не пожизненно.
```

**Полная карта экономики: ECONOMY_ARCHITECTURE.md** (слои 1–7, поток рана, владение)

## 9. СЕМЕЙНЫЙ АЛЬБОМ — АРХИТЕКТУРА ПАМЯТИ (Спринт 43)

Воспоминания теперь **не удаляются** — уходят глубже.

```
decay_sensory() → архивирует в {агент}/archive/memories_YYYY_MM.jsonl
dig_archive(agent_id, query) → Оле ищет в архиве по смыслу
handle_memory_request(agent_id, agent_response) → парсит MEMORY_REQUEST из ответа агента
```

**Как агент вспоминает:**
Агент в любом месте (работа/дом/таверна) может написать:
```
MEMORY_REQUEST: <запрос>
```
Оле поднимет из архива. Результат попадает в контекст следующего шага.

**Правило:** архив не льётся автоматом — только по запросу агента.
