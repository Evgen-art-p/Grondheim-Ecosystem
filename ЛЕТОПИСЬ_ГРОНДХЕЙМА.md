# ЧАСТЬ I · ДНО (вечное, не меняется от спринта к спринту)
═══════════════════════════════════════════════════════════════

## 1. Фрактал из одной единицы

Грондхейм — не программа с модулями. **Фрактал из одной единицы**,
повторённой на всех масштабах без изменения формы:

```
агент → цех → студия/биржа → квартал → город → города
```

Каждое звено — **самостоятельная единица** (житель). У каждого: род, маска,
память, связи, вход↔выход. Один шаблон, вложенный в себя. Строим по
кирпичикам, по одному, до зелёного. Кирпич должен стоять сам.

```
ЖИТЕЛЬ = паспорт(род-якорь) + память + архив
       + работа(маска) + жильё(место) + связи + вход↔выход
```

## 2. Род и Маска — три разные оси

**Род** — дно. Кто единица ДО всякого дела. **Маска** — профессия/роль,
надетая на род. Сменная линза. Снимается — род остаётся.

```
КТО он     → паспорт (род-якорь). Неизменное дно.
КЕМ работает → маска (роль). Сменная линза.
ГДЕ живёт   → жильё. Ковчег → своё.
```

## 2.2. Маска = роль в локации (выстрадано 30.06)

Итог Шефа дословно: **«маска одна на лице… просто роль в локации».**
НЕ папки дом/работа/улица наперёд (Брат придумал — отвергнуто), НЕ привязка
к каждой из 15 локаций. Маска — **тип роли**: кем житель ЯВЛЯЕТСЯ здесь:
```
Лока в Студии   → роль работника
Лока в Высотке  → роль хозяйки дома
Лока в Таверне  → роль гостьи
Лока без всего  → ГОЛАЯ (родилась, но никем в городе не является)
```
Масок НЕМНОГО (роли), локаций МНОГО (15). Маска = «кем я тут», локация =
«где я». **Маска к прогулкам и памяти мест ОТНОШЕНИЯ НЕ ИМЕЕТ** — память
«где был/где сейчас» это sensory, не маска. Не путать (Брат путал весь день).

## 2.3. Локация = самостоятельная единица (закреплено 30.06)

**Локация — недвижимое. Единица с паспортом, как житель, но НЕ дышит:**
нет движка, 4 слоёв, маски. Плоский паспорт — один файл.
```
ЛОКАЦИЯ → passport.json (место вообще: стены, вид, атмосфера)
ЖИТЕЛЬ  → папка: passport + core/resonance/sensory/archive + маски/ (дышит)
```
Закон Движимого/Недвижимого: житель НИКОГДА не лежит внутри локации —
только ссылкой. «Где житель» = последняя запись в sensory.

**Иерархия города (4 уровня):**
```
ГРОНДХЕЙМ (город 0000 — сам холст)
   РАЙОНЫ: Деловой центр · Портовый узел · Храмовый комплекс (поле District)
   КВАРТАЛ = САМ объект-локация вида «жильё», НЕ контейнер, НЕ поле.
      Торговый Квартал — локация рядом с Биржей, со своим прямоугольником.
   ЛОКАЦИИ: Студия, Биржа, Маяк, Высотка, Гексагон...
```
Поле «Квартал» УБРАНО из формы локации. Осталось «Район». Соцпрофиль места
(Social_Rank/Profession/Area/Access) — в паспорте ЛОКАЦИИ.

## 2.4. ДОМ = диалог, лечащий стресс. Житель носит дом В СЕБЕ (закреплено 30.06)

**Прописка = ДОМ** (где спит, ест, отдыхает, СНИЖАЕТ плохой стресс). Одна.
Оформляет **Брат в кабинете**.

**Дом ≠ экран. Дом — часть личности, носимая ВСЕГДА.** Слова Шефа:
**«где бы он ни был — не должен себя забывать… помнить, что есть дом».**
```
житель ВСЕГДА несёт в себе:
   КТО Я (личность)        — паспорт
   ГДЕ МОЙ ДОМ + ЧТО ТАМ   — «домашний промпт» (моя кофта, мой Маяк за окном),
                             ЛИЧНОЕ описание дома, отдельное от паспорта локации
   ГДЕ Я СЕЙЧАС            — last_location (sensory)
```
Домашний промпт = старый «домашний промпт» студии `-2`. Живёт в ЖИТЕЛЕ
(субъективное чувство дома). Переход ковчег→дом = смена СОСТОЯНИЯ жителя:
был бездомный (Лока сейчас, `прописка:null`) → Брат прописал → завёлся
домашний промпт → житель ВЕЗДЕ помнит дом. Чат не меняет экран — меняется
сам житель.

**Дом — место ДИАЛОГА** (как cabinet_chat `-2` снижал стресс). Живой
разговор, что лечит. Образец `soul_tools.py`/`prompts.py` из `-2`.

**РАБОТА — другое, сложнее:** id + должность + пайплайны (машина труда).
Две РАЗНЫЕ привязки:
```
РАБОТА → дело: id + должность + пайплайн
ДОМ    → отдых: сон, еда, снятие стресса, ДИАЛОГ + «не забывать себя»
```

## 3. Заряд (то, что звали стрессом)

Раздражитель ±, маятник к нулю-балансу. Полярность −1..+1. Не копится —
уравновешивается. Сила = глубина касания по воронке памяти.

## 4. Воронка памяти + четыре слоя

```
маяки/метки → sensory ⚡ · маски → маски/ · связи → resonance 🩸
род → core 🦴 (=паспорт) · тишина → archive 🌑 (у Оле)
```
Гравитация: вес = резонанс_с_родом × резонанс_с_маской × плотность_связей.

## 5. Тело носит лёгкое · Чувство не отчёт

Единица не таскает бездну. Тело — лёгкое (паспорт, свежесть, адреса).
Глубина — у хранителя, срезом по запросу. Заряд меняет ЧТО доступно
(какой слой открыт), не строчку. Чувство = сдвиг доступной памяти.

## 6. Память: рамка одна, наполнение разное

TURBO — лента (RAM). TRADING — архив с историей. LIVING_BOOK — рукопись.
Образец «руки в долгую память» — trading (remind/remember).
