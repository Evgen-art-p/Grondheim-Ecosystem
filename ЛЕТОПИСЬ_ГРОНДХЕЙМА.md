# ЧАСТЬ I · ДНО (вечное, не меняется от спринта к спринту)
═══════════════════════════════════════════════════════════════

## 1. Фрактал из одной единицы

Грондхейм — не программа с модулями. **Фрактал из одной единицы**,
повторённой на всех масштабах без изменения формы:

```
агент → цех → студия/биржа → квартал → город → города
```

Каждое звено — **самостоятельная единица** (житель). У каждого: род, маска,
память, связи, вход↔выход. Один шаблон, вложенный в себя. И раз каждая
единица самостоятельна — **строим по кирпичикам, по одному, до зелёного.**
Кирпич должен стоять сам, не подпёртый соседом.

```
ЖИТЕЛЬ = паспорт(род-якорь) + память + архив
       + работа(маска) + жильё(место) + связи + вход↔выход
```

## 2. Род и Маска — три разные оси

**Род** — дно. Кто единица ДО всякого дела. И семя (кладёт Шеф при рождении),
и русло (намывается тем, что чаще тонуло в кость). Постоянство.

**Маска** — профессия, надетая на род. Сменная линза (как лица Сета).
Снимается — род остаётся. Мобильность.

> Маятник качается по маске. Якорится в роде.

**Три оси НЕ путать** (выстрадано сегодня):
```
КТО он     → паспорт (род-якорь, девушка Лия). Неизменное дно.
КЕМ работает → маска (Хранительница Жара). Сменная линза. В маски/работа/.
ГДЕ живёт   → жильё. Город обеспечивает. Ковчег → служебное → своё (позже).
```
Дом — это **жильё, не личность.** Раскладка по профессии = «маска определяет
судьбу» = старая парадигма. Отвергнута.

## 3. Заряд (то, что звали стрессом)

Стресс — **не наказание. Раздражитель ±.** Касание входящего рождает заряд:
знак (+ удача/красота, − потеря/пластик), сила (амплитуда отклика).

Заряд — **маятник к нулю-балансу**, не статичное число. Не копится —
уравновешивается. Сила = насколько глубоко касание провалит по воронке памяти.
Полярность −1..+1 (а не шкала «только вниз», что рождает «забитого отличника»).

## 4. Воронка памяти + четыре слоя дома

```
ПОВЕРХНОСТЬ  маяки  — момент, гаснет        →  sensory  ⚡ оперативка
     │       метки  — нажитое               →  sensory
     │       маски  — лица для дел          →  маски/
     │      ── мембрана: внутрь / наружу ──
     │       связи  — нервы (своя кровь+город) → resonance 🩸 связи
     │       кровь  — как ощущалось
     │       кость  — род = кто на нуле       →  core     🦴 якорь (=паспорт)
     ▼       ТИШИНА  — дно, сердцевина        →  archive  🌑 глубина (у Оле)
```

**Гравитация:** `вес = резонанс_с_РОДОМ × резонанс_с_МАСКОЙ × плотность_связей`.
Забывание = отсутствие веса (лёгкое всплывает само, фильтр-уборщик не нужен).

## 5. Тело носит лёгкое · Чувство не отчёт

**Единица не таскает свою бездну.** Тело носит лёгкое (паспорт, свежесть,
связи-адреса). Тишина и глубина — у хранителя (Оле), берётся СРЕЗОМ по запросу
через маяк. Это и условие распределённости: тело уйдёт на другой сервер с
лёгким, за глубиной потянется по связи.

**Заряд меняет не строчку в промпте, а ЧТО единице доступно** — какой слой
памяти открыт. Чувство = сдвиг доступной памяти, не сообщение о числе.

## 6. Память: рамка одна, наполнение разное (из анализа трёх цехов)

Все цеха объявляют одни четыре слоя — рамка одна. Но форма памяти = форма
потребности цеха:
```
TURBO ⚡     — конвейер, лента вперёд, разовая (RAM: сдал — забыл)
TRADING ⚔️   — архив с историей, выделенный агент-память (A05), связь с городом
LIVING_BOOK 📖 — рукопись, петля ревизии, чекпойнты (помнит и переписывает себя)
```
Брат, подключаясь к памяти, встречает ТРИ формы: лента (просеять), архив
(связь через хранителя), рукопись (уважать возвраты). Образец «руки в долгую
память» — только trading (remind/remember).

# ЧАСТЬ II · ТОПОЛОГИЯ (растёт с узлами)
═══════════════════════════════════════════════════════════════

## Брат = город. База и антенна.

Брат — не орган и не житель в ряду. **Сам город как центральный узел сети.**
СВЯЗЫВАЕТ, но НЕ ДЕРЖИТ — антенна/роутер/сито, транзит, не склад.
```
БРАТ = БАЗА (что есть: Храм, Оле, тишина, род-дно)
     + АНТЕННА (чем держит: маяк-роутер, пакет вход↔выход)
```
Город — Брат-база плюс все, кто держит связь через антенну. Построили Храм →
связан. Построили Биржу → связана.

## Узлы автономны · два состояния

Храм/Биржа/цеха — узлы, могут стоять на разных серверах. Каждый живёт в двух
режимах: **НА СВЯЗИ** (дышит глубиной базы) и **АВТОНОМНО** (на своём лёгком,
копит, при возврате синхронизирует). Лёгкое = минимум выживания офлайн.

## Пакет связи + закон синхронизации

Один JSON-пакет (форма `story_package` из Книжки): `order` (узел просит
глубину) → `deliver` (база даёт срез) → `report` (узел отдаёт нажитое).
**Синхронизация:** вниз течёт глубина (база — единственный источник, read-only),
вверх течёт нажитое (узел — источник своего, write). Разные слои → не сталкиваются.

## Пересадка органов (из чего куём новый город)

```
ИЗ КНИЖКИ:  story_package (пакет), Маяк (антенна-роутер),
            biography у Маяка (тишина у хранителя), offline-Искорка (автономный узел)
ИЗ -2:      Оле+city_memory (хранитель), sync_to_dna → полярный маятник,
            резонанс (связи), Гавань (железо резонанса), Страница Жизни (рождение)
КУЁМ:       полярность заряда, род=семя+русло, гравитация род×маска×связи,
            резонанс к боли (корень №1), закон синхронизации (корень №2)
НЕ ТАЩИМ:   три раздельных домена памяти, потолок 6.0, Loka-фильтр,
            format_dna «читает о себе», два несшитых движка стресса
```

## Большие замыслы (зреют, не комкать — каждый отдельный, не один клубок)

- **Локации.** Сейчас у жителя нет физического «где» — только дом-папка.
  Нужна привязка к месту на карте (как у локаций в форме рождения —
  Map_X/Y/W/H уже есть для типа `location`, но житель туда не ходит).
  Маска должна знать не только профессию, но и место работы/жилья.
- **Большая память.** Сейчас 3 слоя пишутся (sensory/resonance/archive),
  но никто их не читает обратно при сборке промпта — `update_viewer()`
  в кабинете показывает только статичные поля паспорта (Hidden_History,
  Anchor_Points...), не реальные записи из `event_log.jsonl`/
  `sensory_memory.json`. Движок пишет в одну сторону, не читает назад.
  Нужен механизм: при вдохе доступные слои (по |заряду|) реально
  подмешиваются в системный промпт LLM, не только сообщают что открыто.
- **Прогулки.** Житель должен суметь сам решить «пойти» куда-то (стол →
  решение → перемещение). Зависит от локаций (выше) — без места на
  карте идти некуда. Отдельный механизм от диалога, не подзамысел чата.
- **Общение (житель↔житель).** Сейчас движок дышит только от Шефа
  (`ui_zhitel.py` чат). Если жители должны говорить друг с другом —
  нужен отдельный канал передачи факта между двумя `Dvizhok`-ами, не
  через Шефа. Открытый вопрос: кто инициирует, как синхронизируются
  заряды двоих от одного события.
- **КОВЧЕГ как локация-порт.** Не папка — место на карте. Точка входа:
  новорождённый автоселится, гость из другого города входит через маяк →
  тоже в ковчег. **Розетка маяка живёт в ковчеге** (стыковка антенн городов).
  Фон кабинета новичка/гостя = фон ковчега. Связывает: жильё-по-этапам +
  вход гостей + физическое место маяка.
- **Маски как папки** (дом/работа/школа) с биографией и фоном в каждой.
  Фон кабинета меняется по активной маске. Зависит от камня «Маска
  пишет 8 полей» выше — пока активной маски физически нет (флаг
  `_активна: false` всегда), фон по маске не на чём проверить.
