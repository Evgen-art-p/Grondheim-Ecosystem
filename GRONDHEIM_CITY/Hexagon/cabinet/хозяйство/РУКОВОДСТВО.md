## 1. СТРУКТУРА СИСТЕМЫ

Храм состоит из двух миров:

**Студия «Шесть Пальцев»** — рабочие инструменты. Здесь персонажи работают с проектами, бизнесом, сценариями. Это внешний мир.

**Храм** — внутренний мир. Здесь те же персонажи работают с человеком. С его жизнью, страхами, правдой.

Большинство персонажей существуют в обоих мирах. У каждого — своя история, которая объясняет как он пришёл в Храм.

## 3. КАК РАБОТАЕТ МАРКЕР [ТЫ НЕ ОДИН]

Это технический триггер выхода из Храма. Единый для всех персонажей.

**Формат у всех хранителей:**
```
[Финальное слово]. Ты не один. [ТЫ НЕ ОДИН]
```

**Как это работает в коде:**
1. Хранитель вставляет `[ТЫ НЕ ОДИН]` в конец своего ответа
2. `guardian.js` ищет `CONFIG.releaseMarker` в ответе
3. Маркер вырезается из видимого текста (человек его не видит)
4. Запускается `_triggerRelease()` → переход в состояние `RELEASE`
5. Играет release-видео → появляется экран выхода

**Важно:** После маркера промпт запрещает писать что-либо. Это критично — иначе триггер не сработает.

### `fire.js` — нужно небольшое обновление
Сейчас промпт Костра хардкожен в `config.js` как строка `fireSystemPrompt`.
После того как `fire_prompt.txt` появится в папке — нужно изменить `_buildSystemPrompt()`:

**Было** (в `config.js`):
```javascript
fireSystemPrompt: `Ты — Костёр...` // длинная строка
```

**Станет** (в `fire.js`, метод `_buildSystemPrompt()`):
```javascript
_buildSystemPrompt() {
  // Загружаем промпт из файла (уже есть метод _loadPrompt в Guardian — можно переиспользовать)
  let prompt = this.systemPrompt; // загружается через _loadPrompt при enter()

  const fireCtx = memory.getFireContext();
  if (fireCtx) {
    prompt += `\n\nПАМЯТЬ: ${fireCtx}`;
  }

  if (this.pendingGuardian) {
    prompt += `\n\nВАЖНО: Человек уже произнёс ключевое слово. Заверши разговор когда почувствуешь момент. Скажи последние слова и добавь в конец: [ТЫ НЕ ОДИН]. 1-3 сообщения максимум.`;
  }

  return prompt;
}
```

И добавить загрузку промпта в метод `enter()`:
```javascript
enter() {
  this.history = [];
  this.released = false;
  this.isWaiting = false;
  this.pendingGuardian = null;
  this.systemPrompt = ''; // добавить поле
  if (this.outputEl) this.outputEl.innerHTML = '';

  // Загрузить промпт Костра
  this._loadPrompt('3_koster/fire_prompt.txt').then(text => {
    this.systemPrompt = text;
  });

  const fireCtx = memory.getFireContext();
  if (fireCtx) memory.markReturnToFire();

  threshold.start();
}
```

## 5. ЛОГИКА ПЕРЕХОДОВ В ХРАМЕ

```
ВХОД ("ищи") 
    ↓
ТЕМНОТА + ГОЛОС "Проснись"
    ↓
БЕЗЛИКИЙ — спрашивает "Что ты ищешь?"
    ↓
    ├── Ключевое слово найдено → ХРАНИТЕЛЬ
    └── Не знает → КОСТЁР
                      ↓
                      Человек проговаривает себя
                      Ключевое слово найдено (pendingGuardian)
                      Костёр чувствует момент → [ТЫ НЕ ОДИН]
                      ↓
                  ХРАНИТЕЛЬ
                      ↓
                  Диалог (3-4 реплики)
                  Маркер [ТЫ НЕ ОДИН]
                      ↓
                  RELEASE видео
                      ↓
                  ВЫХОД — луч света, "Ты здесь"
```

## 6. ПАМЯТЬ МЕЖДУ ВИЗИТАМИ

Храм помнит человека через `localStorage`.

| Что помнит | Где хранится | Как используется |
|-----------|-------------|-----------------|
| Был ли раньше | `temple_visited` | Безликий говорит "Ты уже был здесь. Что теперь?" |
| Сколько раз | `temple_visits` | Статистика |
| К кому ходил | `temple_guardian_log` | Потенциально — персонализация |
| Темы Костра | `temple_fire_topics` | Костёр помнит о чём говорили |
| История Костра | `sessionStorage` | Контекст если вернулся к Костру в той же сессии |

## 7. API И БЕЗОПАСНОСТЬ

Сейчас API ключ открыт в `config.js`. Пока проект локальный — не критично.

**Когда будешь выкладывать в интернет — обязательно:**
1. Вынести ключ на серверный прокси
2. Убрать `apiKey` из клиентского кода
3. Запросы делать через свой бэкенд

Модель сейчас: `deepseek/deepseek-chat` через OpenRouter.
