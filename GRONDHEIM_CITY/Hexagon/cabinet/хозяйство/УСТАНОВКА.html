Карта файлов

Всего 8 файлов. Три категории действий:

NEW — новый файл REPLACE — заменить целиком APPEND — добавить в конец
js/session.js NEW Новый файл. Положить в папку js/
js/altars.js NEW Новый файл. Положить в папку js/
js/ear-agent.js NEW Новый файл. Положить в папку js/
js/scroll.js NEW Новый файл. Положить в папку js/
js/main.js REPLACE Заменить старый main.js
js/fire.js REPLACE Заменить старый fire.js
js/guardian.js REPLACE Заменить старый guardian.js
css/temple.css APPEND Открыть temple.css, вставить scroll.css в самый конец

Пошаговая установка

01
Сделай резервную копию

Скопируй оригинальные main.js, fire.js, guardian.js куда-нибудь. На случай если что-то пойдёт не так.

02
Положи 4 новых файла в js/
temple/
└── js/
    ├── session.js      ← новый
    ├── altars.js       ← новый
    ├── ear-agent.js    ← новый
    ├── scroll.js       ← новый
    ├── main.js         ← заменить
    ├── fire.js         ← заменить
    └── guardian.js     ← заменить
03
Замени main.js, fire.js, guardian.js

Просто перетащи новые файлы поверх старых. Они содержат оригинальный код + вставки для Свитка (помечены комментарием ← ХРАМ SCROLL).

04
Добавь стили в css/temple.css

Открой temple.css. Прокрути в самый конец. Вставь всё содержимое файла scroll.css.

⚠️ Не заменяй temple.css целиком — только добавь в конец.
05
Проверь config.js

Ухо-Агент использует те же настройки API что и весь Храм — из CONFIG.api. Больше ничего менять не нужно.

✓ Твой config.js уже совместим. api.endpoint, api.apiKey, api.model — всё читается автоматически.
06
Открой через Live Server и проверь

Пройди полный путь: «ищи» → Безликий → Хранитель → скажи «спасибо». Должен появиться лоадер «Храм ткёт твой Свиток...», затем Свиток.

Отладка

Смотри консоль браузера (F12)

Ключевые логи которые должны появиться при прохождении:

[Храм] Сессия: initialKey="...", mood=cold          ← Безликий зафиксировал слово
[Храм] Костёр: ключевое слово → finch               ← ключевое слово найдено
[Храм] Ухо-Агент: запуск синтеза {...}              ← guardian.js передал сессию
[УХО-АГЕНТ] Алтарь: earth | Человек ищет структуру ← агент выбрал алтарь

Если Свиток не появляется

Проверь в консоли — скорее всего ошибка API. Самые частые причины:

API 401  → неверный ключ в config.js
API 429  → превышен лимит запросов
JSON parse error → модель вернула не JSON (попробуй другую модель)

Тест без прохождения всего пути

В консоли браузера можно вызвать агента напрямую:

import('./js/ear-agent.js').then(m => 
  m.synthesizeScroll({
    initialKey: 'я потерял смысл',
    initialMood: 'cold',
    pathType: 'through_fire',
    guardianId: 'finch',
    fireDialogue: [
      {role:'user', content:'мне пусто внутри'},
      {role:'assistant', content:'пустота — это место для нового'}
    ],
    keeperDialogue: [
      {role:'user', content:'хочу найти смысл'},
      {role:'assistant', content:'смысл не ищут — его создают'}
    ]
  }).then(console.log)
)

Сброс памяти для чистого теста

import('./js/memory.js').then(m => m.memory.clear())

Или просто: localStorage.clear() в консоли.

Как это работает (кратко)

«ищи» → Безликий
        ↓
        session.initialKey = первое слово
        session.initialMood = cold|hot|confused

        ↓ развилка ↓

   КОСТЁР                    НАПРЯМУЮ
   fire.js                   (fireDialogue = [])
   каждая реплика            
   → session.fireDialogue[]

        ↓ оба пути сходятся ↓

   ХРАНИТЕЛЬ
   guardian.js
   каждая реплика → session.keeperDialogue[]

        ↓ маркер [ТЫ НЕ ОДИН] ↓

   _launchScrollPhase()
   → session.guardianId, session.pathType
   → showScrollLoading()
   → synthesizeScroll(session)   ← один API-вызов
   → showScroll(data)
   ↓
   СВИТОК (4 блока + промпт)
   [скопировать] [забрать в мир → PNG] [уйти]
        ↓
   RELEASE → EXIT
