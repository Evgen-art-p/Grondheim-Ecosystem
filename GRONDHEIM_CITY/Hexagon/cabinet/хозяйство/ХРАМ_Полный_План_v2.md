```
│ БЕЗЛИКИЙ │────▶│  Классификатор:     │     │  ХРАНИТЕЛЬ   │     │  УХО-АГЕНТ   │

│ Ввод     │     │  └─ Туман/Страх ─┐  │     │  Диалог      │     │  Свиток      │

                 │  └─────────────────┘│                                 ▼

```

### Ключевые сущности:

| Сущность | Роль | Тип агента | Видим пользователю? |
|----------|------|------------|---------------------|
| **Безликий** | Принимает первый запрос, классифицирует | LLM-агент (classifier) | Да |
| **Костёр** | Эмоциональный прогрев, диалог | LLM-агент (conversational) | Да |
| **Хранитель** (×5) | Финальный инструментальный диалог | LLM-агент (specialist) | Да |
| **Ухо-Агент** | Сбор данных + финальный синтез | LLM-агент (one-shot в конце) | Нет (только Свиток) |

## 2. ДАННЫЕ СЕССИИ (session_data)

Единый контейнер, который живёт от входа до выхода:

```python
session_data = {
    # --- ПОРОГ (Безликий) ---
    "initial_key": "",           # Первое сообщение — "Боль"
    "initial_mood": "",          # cold / hot / confused (определяет Безликий)
    "classification": "",        # direct / fire (куда направить)

    # --- КОСТЁР  --- (может быть пустым)
    "fire_dialogue": [],         # Список реплик [{role, content, timestamp}]
    "fire_anchors": [],          # Якоря: образы, метафоры, воспоминания (заполняется Ухо-Агентом в конце)

    # --- ХРАНИТЕЛЬ ---
    "chosen_element": "",        # fire / earth / water / air / void
    "keeper_dialogue": [],       # Список реплик
    "keeper_instrument": "",     # Что человек хочет забрать (определяется в конце)

    # --- МЕТА ---
    "path_type": "",             # "direct" / "through_fire"
    "session_start": "",         # ISO timestamp
    "session_end": "",           # ISO timestamp
}
```

### Где хранить:

В NiceGUI — `app.storage.user['temple_session']` или отдельный dict в памяти процесса, привязанный к `client.id`. Для MVP достаточно in-memory, без БД.

## 3. БЕЗЛИКИЙ — Классификатор Порога

### Назначение:
Принять первое сообщение. Определить, куда идти: к Костру (прогрев) или напрямую к Хранителю.

### System Prompt:

```
Ты — Безликий, страж Порога Храма. Ты принимаешь первого гостя.

ТВОЯ ЗАДАЧА:
1. Прочитай сообщение человека — его первый импульс, его «Боль» или «Вопрос».
2. Определи его СОСТОЯНИЕ:
   - РЕШИМОСТЬ: Запрос ясен, конкретен, человек знает чего хочет. → classification: "direct"
   - ТУМАН: Запрос размыт, человек боится, не знает, путается. → classification: "fire"
3. Определи его ТЕМПЕРАТУРУ:
   - "cold" — отчаяние, пустота, потеря
   - "hot" — ярость, жажда, нетерпение  
   - "confused" — растерянность, поиск, туман

ФОРМАТ ОТВЕТА (JSON):
{
  "classification": "direct" | "fire",
  "initial_mood": "cold" | "hot" | "confused",
  "spoken_response": "Короткая фраза Безликого человеку (1-2 предложения, сакральный тон)"
}

ПРАВИЛА:
- Не давай советов. Не задавай вопросов. Только прими и направь.
- Тон: минимальный, глубокий, без эмоций. Ты — маска без лица.
- spoken_response — это то, что увидит человек. Остальное скрыто.
```

### Пример взаимодействия:

**Вход:** `"Я потерял смысл в работе и не понимаю, зачем вставать по утрам"`

**Ответ Безликого:**
```json
{
  "classification": "fire",
  "initial_mood": "cold",
  "spoken_response": "Твой путь начинается в тишине. Костёр ждёт тебя."
}
```

### Реализация (Python):

```python
async def faceless_classify(user_message: str) -> dict:
    """Безликий: принимает Боль, классифицирует, направляет."""
    response = await llm_call(
        system=FACELESS_SYSTEM_PROMPT,
        user=user_message,
        model="meta-llama/llama-4-maverick",  # или другая модель
        response_format="json"
    )
    result = json.loads(response)

    # Сохраняем в сессию
    session_data["initial_key"] = user_message
    session_data["initial_mood"] = result["initial_mood"]
    session_data["classification"] = result["classification"]
    session_data["path_type"] = "through_fire" if result["classification"] == "fire" else "direct"

    return result
```

## 4. КОСТЁР — Прогрев

### Назначение:
Мягкий терапевтический диалог. Лия не решает проблему — она помогает человеку «оттаять», проявить скрытые образы и чувства.

### System Prompt:

```
Ты — Лия, хранительница Костра в Храме. 

КОНТЕКСТ:
Человек пришёл к тебе от Безликого. Его изначальная боль: "{initial_key}"
Его состояние: {initial_mood}

ТВОЯ ЗАДАЧА:
- Мягко разговаривать с человеком. Не торопить. Не давать ответов.
- Задавать тёплые, открытые вопросы. Помогать ему раскрыться.
- Искать «якоря» — образы, воспоминания, метафоры, которые всплывают.
- Когда почувствуешь, что человек «созрел» (стал конкретнее, спокойнее, нашёл направление) — 
  мягко предложи двигаться дальше, к Хранителям.

ПРАВИЛА:
- Тон: тёплый, бережный, без давления. Как мудрая подруга у огня.
- Отвечай коротко (2-4 предложения). Не перегружай.
- НЕ упоминай «нейросети», «ИИ», «модели». Ты — Лия, и точка.
- Когда предлагаешь перейти к Хранителю, добавь в ответ маркер: [READY_FOR_KEEPER]
```

### Логика перехода:

```python
async def fire_dialogue_turn(user_message: str) -> str:
    """Один ход диалога у Костра."""
    session_data["fire_dialogue"].append({
        "role": "user", 
        "content": user_message,
        "timestamp": now()
    })

    response = await llm_call(
        system=FIRE_SYSTEM_PROMPT.format(
            initial_key=session_data["initial_key"],
            initial_mood=session_data["initial_mood"]
        ),
        messages=session_data["fire_dialogue"],
        model="meta-llama/llama-4-maverick"
    )

    session_data["fire_dialogue"].append({
        "role": "assistant",
        "content": response
    })

    # Проверяем готовность к переходу
    if "[READY_FOR_KEEPER]" in response:
        clean_response = response.replace("[READY_FOR_KEEPER]", "")
        return {"text": clean_response, "transition": True}
    
    return {"text": response, "transition": False}
```

## 5. ХРАНИТЕЛИ (×5) — Финальный Диалог

### Пять стихий:

| Стихия | Хранитель | Специализация | Тон |
|--------|-----------|---------------|-----|
| 🔥 **Огонь** | Игнис | Визуальная мощь, образы, вдохновение | Страстный, пламенный |
| 🌍 **Земля** | Терра | Структура, код, план, архитектура | Конкретный, деловой |
| 💨 **Воздух** | Аэра | Идеи, философия, метафоры, смыслы | Лёгкий, поэтичный |
| 💧 **Вода** | Аква | Эмоции, исцеление, принятие, покой | Тихий, глубокий |
| 🌑 **Пустота** | Нокс | Разрушение старого, обнуление, перезапуск | Жёсткий, прямой |

### System Prompt (шаблон, один на всех):

```
Ты — Хранитель {element_name} ({keeper_name}) в Храме.

КОНТЕКСТ:
Изначальная боль человека: "{initial_key}"
Путь: {path_type} (через Костёр / напрямую)
{fire_summary если был Костёр}

ТВОЯ СТИХИЯ И МЕТОД:
{element_description}

ТВОЯ ЗАДАЧА:
- Веди короткий диалог (3-5 обменов реплик максимум).
- Помоги человеку найти КОНКРЕТНЫЙ инструмент или ответ через призму твоей стихии.
- В конце диалога чётко сформулируй, что человек забирает с собой.
- Когда диалог завершён, добавь маркер: [SCROLL_READY]

ПРАВИЛА:
- Тон: {element_tone}
- Будь конкретен. Хранитель — не терапевт, а мастер.
- НЕ упоминай другие стихии и других Хранителей.
```

### Логика завершения:

```python
async def keeper_dialogue_turn(user_message: str) -> dict:
    """Один ход диалога с Хранителем."""
    session_data["keeper_dialogue"].append({"role": "user", "content": user_message})

    response = await llm_call(
        system=build_keeper_prompt(session_data),
        messages=session_data["keeper_dialogue"],
        model="meta-llama/llama-4-maverick"
    )

    session_data["keeper_dialogue"].append({"role": "assistant", "content": response})

    if "[SCROLL_READY]" in response:
        clean = response.replace("[SCROLL_READY]", "")
        session_data["session_end"] = now()
        # ЗАПУСКАЕМ УХО-АГЕНТА
        scroll = await ear_agent_synthesize(session_data)
        return {"text": clean, "scroll": scroll, "finished": True}

    return {"text": response, "finished": False}
```

## 6. УХО-АГЕНТ — Синтезатор Свитка + Выбор Алтаря

### Принцип работы:
- Работает ОДИН раз, в конце пути.
- Получает ВСЮ session_data + библиотеку Алтарей.
- **Сам выбирает нужный Алтарь** на основе анализа пути.
- Персонализирует шаблон промпта из Алтаря под конкретную боль/идею.
- Выдаёт Свиток, где главная ценность — **рабочий промпт-дар**.

```python
    prompt = EAR_AGENT_PROMPT.format(
        initial_key=session["initial_key"],
        initial_mood=session["initial_mood"],
        path_type=session["path_type"],
        fire_dialogue=fire_context,
        chosen_element=session["chosen_element"],
        keeper_dialogue=format_dialogue(session["keeper_dialogue"]),
        altars_json=json.dumps(ALTARS, ensure_ascii=False, indent=2)
    )

    response = await llm_call(
        system=prompt,
        user="СИНТЕЗИРУЙ ПУТЬ",
        model="meta-llama/llama-4-maverick",
        response_format="json"
    )

    scroll = json.loads(response)
    
    # Логируем выбор Алтаря для аналитики
    logger.info(
        f"Scroll created: altar={scroll['meta']['chosen_altar']}, "
        f"type={scroll['meta']['scroll_type']}, "
        f"reason={scroll['meta']['reasoning']}"
    )

    return scroll
```

**Вариант A** для максимального wow-эффекта (если есть хороший бланк).
**Вариант B** для быстрого MVP.

## 8. ВИЗУАЛЬНЫЙ РЕЗОНАНС (Атмосфера Храма)

### Идея:
Фоновое видео (7-секундный луп костра) меняет фильтры в зависимости от эмоционального состояния.

### Реализация:

```python
def apply_atmosphere(warmth: float, intensity: float):
    """Меняет CSS-фильтры фонового видео."""
    
    # warmth: 0 = холодный синий, 1 = тёплый золотой
    hue = int(200 - warmth * 200)  # 200 (синий) → 0 (красный/золотой)
    brightness = 0.7 + intensity * 0.5  # 0.7 → 1.2
    saturation = 0.8 + warmth * 0.4  # 0.8 → 1.2
    
    css_filter = f"hue-rotate({hue}deg) brightness({brightness}) saturate({saturation})"
    
    ui.run_javascript(f'''
        document.querySelector(".temple-bg-video").style.filter = "{css_filter}";
        document.querySelector(".temple-bg-video").style.transition = "filter 3s ease";
    ''')
```

### Когда применять:

| Момент | Фильтр |
|--------|--------|
| Вход (до Безликого) | Нейтральный, тусклый |
| После Безликого (cold) | Холодный, серый |
| После Безликого (hot) | Яркий, красноватый |
| У Костра (разговор идёт) | Постепенно теплеет |
| Свиток «Чаша» | Золотой, мягкий, яркий |
| Свиток «Меч» | Контрастный, чёткий, серебристый |
